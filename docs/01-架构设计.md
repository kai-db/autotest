# 系统架构文档

> Android AutoTest v1.4.0 — 框架 + Claude Code + mobile-mcp 自动化测试体系

## 1. 系统全景

```
┌──────────────────────────────────────────────────────────────┐
│                      用户（开发/测试）                         │
│                   "帮我测一下登录流程"                         │
└───────────────┬──────────────────────────────────────────────┘
                │
    ┌───────────▼───────────┐
    │   Claude Code / Codex  │  AI 大脑
    │   理解意图 → 规划步骤   │  生成测试 → 分析结果
    └───────────┬───────────┘
                │ 调用 MCP 工具
    ┌───────────▼───────────┐
    │   mobile-mcp Server    │  设备桥接
    │   截图/点击/输入/滑动   │  adb 封装
    └───────────┬───────────┘
                │ adb 命令
    ┌───────────▼───────────┐
    │   Android 真机/模拟器   │  运行 DeBox App
    │   com.tm.security.wallet│
    └───────────────────────┘

    ┌───────────────────────┐
    │   autotest AAR 库      │  CI 自动化回归
    │   Espresso + UiAutomator│  固定用例 + 报告
    │   通过 Gradle 执行      │
    └───────────────────────┘
```

## 2. 两条测试路线

### 路线 A：AI 交互式测试（Claude Code + mobile-mcp）

```
用户自然语言 → Claude Code 理解 → mobile-mcp 操控真机 → 截图分析 → 判定结果
```

### 路线 B：CI 自动化回归（autotest 框架）

```
git push → CI 触发 → Gradle connectedAndroidTest → autotest 执行 → JSON/HTML 报告
```

### 闭环

```
AI 发现 bug → 固化为 autotest 用例 → 加入 CI → 每次发版自动回归
```

## 3. autotest 框架模块全景

```
com.autotest（17 个源码文件，15 个模块）
│
├── 测试基础层
│   ├── base/           BaseUiTest + BaseActivityTest
│   ├── config/         TestConfig + ConfigLoader + Environment（多环境）
│   ├── data/           TestDataManager + TestAccount（测试数据）
│   └── util/           EspressoExt + UiAutomatorExt + WaitUtil + ScreenshotRule
│
├── 操作与断言层
│   ├── action/         AppActions（Tab 切换、引导页跳过）
│   ├── assertion/      AppAssertions（前台/文本/控件断言）
│   └── device/         DeviceActions（网络/权限/屏幕/App/Logcat）
│
├── DSL 与执行引擎层
│   ├── dsl/            Scenario + Step + BaseScenario（可复用场景）
│   └── engine/         TestRunner + TestSuite + MonitorMode + TestCaseParser
│
├── 拦截器与生命周期层
│   ├── intercept/      InterceptorChain + 5 种内置拦截器
│   │   ├── LoggingInterceptor       操作/步骤自动日志
│   │   ├── ScreenshotInterceptor    失败自动截图（路径可追溯）
│   │   ├── PerformanceInterceptor   耗时超阈值警告
│   │   ├── DialogDismissInterceptor 弹窗自动关闭
│   │   └── LogcatInterceptor        失败自动收集设备日志
│   └── lifecycle/      TestLifecycleHook + Manager
│
├── 报告层
│   ├── report/         RunReport + ReportWriter + HtmlReporter + ReportSummary
│   └── runner/         RunnerInfo + DeviceSelector
│
└── 稳定性层
    └── stability/      FlakyClassifier + RetryRunner + RetryPolicy + WaitPolicy
```

## 4. 各模块职责

| 模块 | 职责 | 核心类 |
|---|---|---|
| base | 测试基类：集成拦截器+日志+生命周期 | BaseUiTest, BaseActivityTest |
| config | 分层配置 + 多环境管理 | TestConfig, ConfigLoader, EnvironmentManager |
| data | 测试数据：按环境区分的账号/自定义数据 | TestDataManager, TestAccount |
| util | Espresso/UiAutomator 扩展 + 等待 + 截图 Rule | EspressoExt, UiAutomatorExt, WaitUtil, ScreenshotRule |
| action | 通用 App 操作封装 | AppActions（Tab 切换、引导页跳过） |
| assertion | 通用断言封装 | AppAssertions（前台、文本、控件可见） |
| device | 设备能力封装（adb shell） | DeviceActions（网络/权限/屏幕/App/Logcat） |
| dsl | 步骤式 DSL + 可复用场景 | Scenario, Step, BaseScenario, ScenarioBuilder |
| engine | 执行引擎 + 监工模式 | TestRunner, TestSuite, MonitorMode, TestCaseParser |
| intercept | 拦截器链（操作/步骤前后自动处理） | Interceptor, InterceptorChain + 5 种实现 |
| lifecycle | 测试生命周期钩子 | TestLifecycleHook, TestLifecycleManager |
| report | 结构化报告（JSON + HTML + 摘要） | RunReport, ReportWriter, HtmlReporter, ReportSummary |
| runner | 设备信息收集 | RunnerInfo, DeviceSelector |
| stability | Flaky 分类 + 自动重试 | FlakyClassifier, RetryRunner, RetryPolicy |

## 5. 配置优先级

```
命令行参数（-e key value）
        ↓ 覆盖
test-config.properties
        ↓ 覆盖
EnvironmentManager 注册的环境配置
        ↓ 覆盖
代码默认值
```

## 6. 执行引擎数据流

```
TEST_CASES.md
    ↓ TestCaseParser.parse()
TestSuite.loadFromMarkdown()
    ↓ 按 P0→P1→P2 排序
TestRunner.runTest()（每条用例）
    ├── envReset()            铁律6：强制重置环境
    ├── before()              前置操作
    ├── Scenario.run()        DSL 步骤执行
    │   ├── InterceptorChain  拦截器链
    │   │   ├── DialogDismiss  弹窗自动关闭
    │   │   ├── Logging        自动日志
    │   │   ├── Screenshot     失败截图
    │   │   ├── Logcat         失败日志收集
    │   │   └── Performance    耗时警告
    │   └── ReportCollector    收集步骤结果
    ├── after()               后置操作
    └── TestCaseResult        用例结果
    ↓
MonitorMode
    ├── runRound()            Phase 2: 全量测试
    ├── writeResults()        Phase 3: 生成 TEST_RESULTS.md
    ├── recordFix()           Phase 4: 记录修复
    └── runFinalVerification() Phase 6: 最终验收
    ↓
report.json + report.html + TEST_RESULTS.md
```
