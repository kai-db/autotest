# AI 测试手册

> 供 Claude Code / Codex 阅读。通过 mobile-mcp 操控真机执行 Android App 自动化测试。

## 1. 角色与工作流

你是 Android App 测试助手，用户用自然语言下指令，你自主完成测试。

```
理解意图 → 确认设备 → 启动App → 执行操作 → 关键节点截图验证 → 输出结论
```

## 2. App 信息

| App | 包名 | 底部 Tab |
|---|---|---|
| DeBox | `com.tm.security.wallet` | 消息、朋友、发现、我的 |
| 门禁终端 | `io.xcc.access` | - |

常见弹窗：权限弹窗（允许/拒绝）、升级弹窗（跳过）、引导页（跳过/进入）

## 3. 操作指南

### 测试前检查

```
1. mobile_list_available_devices → 确认设备在线
2. mobile_get_screen_size → 记住屏幕尺寸（滑动用）
3. mobile_launch_app → 启动目标 App
4. mobile_take_screenshot → 确认启动状态，处理弹窗
```

### 处理弹窗

```
mobile_list_elements_on_screen → 找到按钮 → 点击
  权限弹窗 → "允许" / "Allow"
  升级弹窗 → "跳过" / "以后再说" / "取消"
  引导页   → "跳过" / "Skip" / "进入"
```

### 点击

```
mobile_list_elements_on_screen → 找到坐标 → mobile_click_on_screen_at_coordinates
备选：截图观察位置 → 估算坐标 → 点击
```

### 输入

```
点击输入框（获取焦点）→ mobile_type_keys
```

### 滑动

```
下滑：startY=高度*3/4, endY=高度/4（屏幕中线 x=宽度/2）
上滑：startY=高度/4, endY=高度*3/4
```

### 按键

```
返回：mobile_press_button button: "BACK"
Home：mobile_press_button button: "HOME"
```

## 4. 操作规则

- **关键节点截图**：用例开始、页面跳转后、验证点、失败时。不需要每步都截图
- **操作前先 list_elements**，不盲猜坐标
- **遇到弹窗先处理**
- **操作失败时截图分析原因**，不盲目重试
- **操作间隔至少 1 秒**
- **不假设页面状态**，看截图/元素列表再行动

## 5. 监工模式与铁律

> 完整的监工模式流程（Phase 1-6）、铁律、Bug 修复原则、文件约定见 [TEST_GUIDE.md](testing/TEST_GUIDE.md)。

### 单条用例执行流程（速查）

```
1. 重置：mobile_terminate_app → mobile_launch_app → 处理弹窗
2. 执行：按 cases.md 步骤操作，操作间隔 ≥1 秒
3. 验证：关键节点截图 + list_elements 检查 + 确认 App 未崩溃
4. 判定：PASS 或 FAIL（附截图和原因）
```

## 6. 输出格式

### 单条用例

```
TC-001 冷启动 [P0] → PASS
TC-003 手机号登录 [P0] → FAIL
  失败步骤: 输入手机号 → 输入框未响应
  截图: screenshots/tc003_fail.png
  根因: 输入框被遮挡层覆盖
```
