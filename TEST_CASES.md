# 测试用例

> 监工模式唯一用例来源。分为框架自身测试和 DeBox App 测试两部分。

---

# 一、框架自身测试

## P0 — 编译与发布

### TC-001 框架编译
- 步骤：执行 `./gradlew :autotest:compileReleaseKotlin`
- 验证：BUILD SUCCESSFUL，无错误

### TC-002 单元测试全部通过
- 步骤：执行 `./gradlew :autotest:test`
- 验证：BUILD SUCCESSFUL，0 failures

### TC-003 发布到 mavenLocal
- 步骤：执行 `./gradlew :autotest:publishToMavenLocal`
- 验证：`~/.m2/repository/com/autotest/autotest/` 下有最新版本 aar 文件

---

# 二、DeBox App 测试

> 包名：com.tm.security.wallet
> 设备：通过 mobile-mcp 连接的 Android 真机
> 底部 Tab：消息、朋友、发现、我的
> 前置条件：设备已解锁、DeBox 已安装、已登录

## P0 — 冒烟测试

### TC-101 冷启动
- 步骤：terminate_app → launch_app com.tm.security.wallet
- 步骤：等待 5 秒
- 步骤：截图确认当前页面
- 验证：App 成功启动，显示消息列表页面（顶部有"DeBox"标题）
- 验证：底部 Tab 栏可见（消息/朋友/发现/我的）

### TC-102 弹窗处理
- 步骤：terminate_app → launch_app
- 步骤：等待 5 秒，截图
- 步骤：如出现弹窗（通知提示/升级弹窗），点击"稍后再说"或"取消"关闭
- 步骤：截图确认弹窗已关闭
- 验证：弹窗正常关闭，App 页面正常显示

### TC-103 消息 Tab（默认 Tab）
- 步骤：启动 App，确认在消息 Tab
- 步骤：截图
- 验证：顶部显示"DeBox"标题
- 验证：子 Tab 可见（全部/私信/群组/俱乐部/DAO）
- 验证：聊天列表有内容（至少显示 1 条会话）

### TC-104 朋友 Tab
- 步骤：点击底部"朋友" Tab
- 步骤：截图
- 验证：顶部显示"朋友"标题
- 验证：页面显示关注/粉丝/新增关注/入群通知/AI助手等入口

### TC-105 发现 Tab
- 步骤：点击底部"发现" Tab
- 步骤：截图
- 验证：页面有功能入口列表
- 验证：页面不是白屏或错误页

### TC-106 我的 Tab
- 步骤：点击底部"我的" Tab
- 步骤：截图
- 验证：显示用户头像或昵称（已登录状态）
- 验证：有设置入口

### TC-107 Tab 来回切换
- 步骤：依次点击 消息 → 朋友 → 发现 → 我的 → 消息
- 步骤：每次切换后等待 2 秒
- 步骤：最后截图确认回到消息 Tab
- 验证：所有切换正常，无白屏/崩溃
- 验证：最终停留在消息 Tab

## P1 — 功能测试

### TC-201 进入群聊
- 步骤：在消息 Tab 点击第一个群聊会话
- 步骤：截图
- 验证：进入群聊页面，顶部显示群名称
- 验证：聊天消息列表可见
- 验证：底部有输入框

### TC-202 群聊返回
- 步骤：从群聊页面按返回键
- 步骤：截图
- 验证：回到消息列表页面

### TC-203 朋友-AI 助手
- 步骤：进入朋友 Tab
- 步骤：点击"AI助手"入口
- 步骤：截图
- 验证：进入 AI 助手页面（显示"DeBox AI"标题）

### TC-204 我的-设置页
- 步骤：进入我的 Tab
- 步骤：点击设置入口（齿轮图标）
- 步骤：截图
- 验证：进入设置页面
- 验证：显示设置项列表

### TC-205 消息-搜索
- 步骤：在消息 Tab 点击搜索图标（放大镜）
- 步骤：截图
- 验证：进入搜索页面或弹出搜索框

## P2 — 稳定性测试

### TC-301 快速 Tab 切换
- 步骤：连续快速切换 Tab 10 次（消息→朋友→发现→我的 循环）
- 步骤：每次间隔 500ms
- 步骤：最后截图
- 验证：App 未崩溃，页面正常显示

### TC-302 后台恢复
- 步骤：启动 App 确认正常
- 步骤：按 Home 键回到桌面
- 步骤：等待 3 秒
- 步骤：重新启动 App（launch_app）
- 步骤：截图
- 验证：App 恢复到之前的页面，无白屏/崩溃
